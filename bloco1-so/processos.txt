# PARTE B — Processo filho
# Cole a saída dos comandos e responda:

PID do filho: 94

cat /proc/[FILHO]/status | grep -E "Pid|PPid|State"
State:  S (sleeping)

ps -o pid,ppid,stat,cmd (antes do kill)

# PARTE C — Questões de síntese

# 1. O campo PPID em /proc/$$/status indica o processo pai do shell.
#    Execute ps -o pid,ppid,cmd e localize o par shell -> processo pai.
#    O que esse encadeamento sugere sobre como os processos sao
#    criados no Linux?

Resposta 1:
Isso mostra que os processos no Linux funcionam como uma “família”, todo processo tem um pai e pode criar filhos. Logo, eles formam tipo uma "árvore", começando do processo inicial.

# 2. O arquivo /proc/<PID>/status mostra o campo State com uma letra
#    (S, R, Z...). Depois de executar kill $FILHO e rodar ps novamente,
#    o que aconteceu com o processo filho? O que isso te faz pensar  
#    sobre a vida de um processo?

Resposta 2:
Depois do kill, o processo filho foi excluido. Isso mostra que processos têm um ciclo: começam, rodam ou esperam e terminam.
